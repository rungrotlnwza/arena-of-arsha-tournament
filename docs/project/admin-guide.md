# คู่มือผู้ดูแลระบบ

เอกสารนี้อธิบายขั้นตอนใช้งานระบบหลังบ้านของรายการแข่งขัน

## หน้าหลังบ้านที่เกี่ยวข้อง

- `/admin/login` : ล็อกอินสำหรับผู้ดูแลระบบ
- `/admin/dashboard` : หน้าสรุปข้อมูล
- `/admin/teams` : หน้าตรวจและจัดการทีม
- `/admin/settings` : หน้าตั้งค่ากิจกรรม
- `/admin/bracket` : หน้าจัดการตารางแข่งขัน

## 1. การเข้าสู่ระบบ

เข้าใช้งานที่ `/admin/login`

ข้อมูลที่ต้องใช้

- `username`
- `password`

หมายเหตุ

- ระบบใช้ JWT cookie สำหรับยืนยันตัวตน
- ถ้า token หมดอายุหรือไม่มีสิทธิ์ ระบบจะพากลับหน้า login

## 2. Dashboard

หน้า `/admin/dashboard` ใช้ดูภาพรวมแบบเร็ว

ข้อมูลที่แสดง

- จำนวนทีมทั้งหมด
- จำนวนทีมที่อนุมัติแล้ว
- จำนวนทีมที่รอตรวจสอบ
- จำนวนที่นั่งคงเหลือเทียบกับ `max_teams`
- รายชื่อทีมสมัครล่าสุด

หน้า dashboard เหมาะสำหรับเช็กสถานะปัจจุบันก่อนเริ่มงานในแต่ละวัน

## 3. การตั้งค่ากิจกรรม

หน้า `/admin/settings` ใช้กำหนดข้อมูลกลางของรายการ

สิ่งที่ตั้งค่าได้

- ชื่อกิจกรรม
- วันที่แข่งขัน
- เวลาแข่งขัน
- สถานที่
- จำนวนทีมสูงสุด `max_teams`
- ลิงก์ถ่ายทอดสด
- วันเวลาเปิดรับสมัคร
- วันเวลาปิดรับสมัคร
- สถานะเปิด/ปิดรับสมัคร

คำแนะนำ

- ควรตั้ง `max_teams` ให้เรียบร้อยก่อนเริ่มจับคู่
- ถ้าเปลี่ยน `max_teams` ขนาด bracket ตั้งต้นจะเปลี่ยนตาม
- ถ้ามีสตรีม ให้ใส่ลิงก์ไว้ล่วงหน้าเพื่อให้หน้า `/live` และคู่แข่งบางคู่ดึงข้อมูลได้

## 4. การตรวจทีมสมัคร

หน้า `/admin/teams` ใช้ตรวจรายชื่อทีมทั้งหมด

ความสามารถหลัก

- กรองสถานะทีม
- ดูข้อมูลทีมแบบละเอียดใน modal
- อนุมัติทีม
- ปฏิเสธทีม

ข้อมูลที่ตรวจได้ใน modal

- ชื่อทีม
- วันเวลาสมัคร
- ช่องทางที่รู้จักกิจกรรม
- ประสบการณ์การแข่งขัน
- ข้อมูลผู้เล่นทั้ง 2 คน
- `Family Name`
- ชื่อตัวละครในเกม
- `Discord ID`

ผลของการอนุมัติ

- ทีมจะไปแสดงที่หน้า `/teams`
- ทีมจะถูกนำไปใช้ใน dropdown สำหรับการจับคู่ในหน้า bracket

## 5. การจัดการ bracket

หน้า `/admin/bracket` ใช้จัดการสายการแข่งขันทั้งหมด

ความสามารถหลัก

- แสดง bracket แบบ pyramid
- แสดง bracket ว่างตามขนาดรายการแม้ยังไม่มีการจับคู่
- ลากเมาส์เพื่อเลื่อนดูพื้นที่ bracket ขนาดใหญ่
- เลือกทีมในแต่ละ slot ผ่าน dropdown
- สร้าง bracket อัตโนมัติจากทีมที่อนุมัติแล้ว
- เลือกผู้ชนะเพื่อดันเข้ารอบถัดไป
- ตั้งสถานะกำลังแข่ง
- ล้างผลการแข่งขัน
- เปิด modal เพื่อแก้เวลาแข่ง, ลิงก์สตรีม, หมายเหตุ หรือจัดการทีมในคู่นั้น
- ลบคู่แข่งขัน
- รีเซ็ตการจับคู่ทั้งหมด

### 5.1 การสร้าง bracket อัตโนมัติ

กดปุ่ม `สร้างตารางอัตโนมัติ`

ผลลัพธ์

- ระบบจะดึงทีมที่มีสถานะ `approved`
- ระบบจะจับคู่รอบแรกให้อัตโนมัติ
- หน้า bracket จะรีเฟรชข้อมูลและแสดงคู่ที่สร้างขึ้น

### 5.2 การจับคู่ด้วยตัวเอง

ถ้ายังไม่ต้องการสร้างอัตโนมัติ สามารถจับคู่เองได้จากช่อง dropdown บนการ์ด

วิธีใช้งาน

1. ไปที่ slot ที่ต้องการ
2. เลือกทีมที่ 1 และทีมที่ 2 จาก dropdown
3. ระบบจะสร้างหรืออัปเดต match ให้ทันที

หมายเหตุ

- หน้า bracket จะแสดง slot ว่างตามขนาดรายการตั้งแต่แรก
- รอบแรกจะขึ้นข้อความแนวว่าให้เลือกทีมเพื่อเริ่มจับคู่

### 5.3 การเลือกผู้ชนะ

ในแต่ละคู่จะมีปุ่มเลือกผู้ชนะของทีมฝั่งซ้ายและขวา

เมื่อเลือกผู้ชนะแล้ว

- สถานะคู่นั้นจะกลายเป็น `completed`
- ผู้ชนะจะถูกส่งเข้าช่องของรอบถัดไปอัตโนมัติ

### 5.4 การเปลี่ยนสถานะระหว่างแข่งขัน

เครื่องมือในแต่ละการ์ดมีดังนี้

- `กำลังแข่ง` : เปลี่ยนสถานะคู่เป็น `ongoing`
- `ล้างผล` : ล้างผู้ชนะและกลับไปสถานะรอแข่ง
- `รายละเอียด` : เปิด modal เพื่อแก้ข้อมูลเชิงลึกของคู่นั้น

### 5.5 การแก้ไขผ่าน modal

modal ของ match ใช้จัดการข้อมูลต่อไปนี้

- รอบการแข่งขัน
- ทีมที่ 1
- ทีมที่ 2
- ทีมที่ชนะ
- สถานะ
- เวลาแข่งขัน
- ลิงก์ถ่ายทอดสด
- หมายเหตุ

นอกจากนี้ยังมีปุ่ม

- `บันทึก`
- `ลบคู่นี้`

### 5.6 การรีเซ็ต bracket

กดปุ่ม `รีเซ็ตการจับคู่`

ผลลัพธ์

- ระบบจะล้างข้อมูลในตาราง `bracket`
- ไม่ลบรายชื่อทีม
- ไม่ลบข้อมูลผู้สมัคร
- ใช้เมื่อต้องการจัดสายใหม่ทั้งหมด

## 6. ลำดับการทำงานที่แนะนำสำหรับแอดมิน

1. ล็อกอินเข้า `/admin/login`
2. ไปที่ `/admin/settings` เพื่อกำหนดค่ากิจกรรมและจำนวนทีมสูงสุด
3. เปิดรับสมัคร
4. ตรวจทีมใน `/admin/teams`
5. อนุมัติทีมที่ผ่านเกณฑ์
6. ไปที่ `/admin/bracket`
7. จับคู่ด้วยตัวเองหรือกดสร้างตารางอัตโนมัติ
8. ระหว่างแข่งขัน อัปเดตสถานะ เวลาแข่ง ลิงก์สตรีม และผู้ชนะ
9. ถ้าต้องจัดใหม่ทั้งสาย ให้ใช้ `รีเซ็ตการจับคู่`

## 7. ข้อควรระวัง

- ปุ่ม `รีเซ็ตการจับคู่` ล้างสายการแข่งทั้งหมด ควรใช้เมื่อแน่ใจว่าต้องจัดใหม่จริง
- การปรับผู้ชนะย้อนหลังอาจทำให้ข้อมูลรอบถัดไปเปลี่ยนตาม
- ควรอนุมัติทีมให้ครบก่อนกด `สร้างตารางอัตโนมัติ`
- ถ้าจะใช้งานสตรีมหน้า `/live` ควรตั้ง `stream_url` ใน settings ให้ถูกต้อง
